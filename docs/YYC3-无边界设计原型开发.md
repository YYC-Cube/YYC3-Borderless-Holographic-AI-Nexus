## 🚀 快速开始

### 前置准备

```bash
# 1. 确保Figma桌面应用已安装并更新
open -a "Figma"

# 2. 启用Figma MCP服务器
# 在Figma中：开发模式 → MCP服务器 → 启用

# 3. 验证MCP服务器运行
curl http://127.0.0.1:3845/mcp

# 4. 配置Claude Code连接Figma MCP
# ~/.claude/settings.json中添加：
{
  "mcpServers": {
    "figma-desktop": {
      "type": "http",
      "url": "http://127.0.0.1:3845/mcp"
    }
  }
}
```

### 基本使用流程

```
1. 在Figma中创建新项目
2. 选择设计元素或框架
3. 复制相应的提示词
4. 在Claude Code中发送提示词
5. Figma AI自动生成或修改设计
6. 查看结果并迭代优化
```

## 🎯 实操案例1：无边界待办事项应用

### 项目概述

创建一个符合YYC³无边界设计理念的待办事项管理应用，支持手势、语音、眼动等多种交互方式。

### 阶段1：需求理解与概念设计

**步骤1：创建Figma项目**

```bash
# 1. 打开Figma
open -a "Figma"

# 2. 创建新项目
# File → New File
# 命名为：Todo-App-Prototype

# 3. 创建画板
# Frame → Desktop Frame (1920 x 1080)
```

**步骤2：使用提示词生成概念设计**

在Claude Code中发送以下提示词：

```
请基于以下需求，创建符合YYC³无边界设计理念的概念设计：

【需求描述】
开发一个待办事项管理应用，支持添加、删除、标记完成、过滤功能，采用无边界设计理念。

【设计要求】
1. 核心理念：无边界核心、无按钮设计、一切依托交互
2. 交互方式：手势、滑动、拖拽、语音、眼动等自然交互
3. 布局原则：动态布局、响应式设计、无固定边框、沉浸式容器
4. 用户体验：沉浸式体验、上下文感知、即时反馈、无边界导航

【技术栈】
- 前端框架：React 18
- 多模态交互：Web Speech API / Hammer.js / Pointer Events
- AI集成：GLM 4.7

【输出要求】
请生成：
1. 概念设计草图（包含主要界面和交互流程）
2. 核心交互组件列表（滑动面板、动态卡片、手势导航等）
3. 设计规范草案（颜色、字体、间距系统）
4. 交互模式说明（手势导航、拖拽排序、语音控制等）
```

### 2. 无边界设计原则应用

**原则1：去按钮化**

```
传统设计：
- 使用"添加"按钮添加待办事项
- 使用"删除"按钮删除待办事项
- 使用"完成"按钮标记待办事项

无边界设计：
- 从底部滑动召唤添加界面
- 左右滑动删除待办事项
- 点击标记完成待办事项
```

**原则2：动态布局**

```
传统设计：
- 固定布局，元素位置和大小固定
- 响应式设计，但在断点处突变

无边界设计：
- 动态布局，元素位置和大小随用户交互而变化
- 流体布局，元素随屏幕尺寸平滑变化
- 模块化组件，根据用户行为动态组合和呈现
```

**原则3：沉浸式体验**

```
传统设计：
- 界面元素多，视觉干扰大
- 用户注意力分散在界面元素上

无边界设计：
- 界面元素少而精，视觉干扰小
- 用户注意力集中在任务本身
- 沉浸式容器，内容占据整个视口
```

## 🔧 常见问题解决

### 问题1：Figma AI无法理解提示词

**症状**：

- Figma AI生成的结果不符合预期
- 生成的设计不符合无边界原则

**解决方案**：

1. **检查提示词是否清晰明确**

   ```
   ❌ 模糊：创建一个好看的界面
   ✅ 具体：创建一个沉浸式的待办事项应用界面，使用蓝色主题，支持手势滑动删除
   ```

2. **提供更多的上下文信息**

   ```
   ❌ 缺少上下文：设计一个按钮
   ✅ 提供上下文：设计一个用于添加待办事项的交互组件，不使用传统按钮，通过手势或语音触发
   ```

3. **分步骤描述需求**

   ```
   ❌ 一次性描述所有需求：创建一个完整的应用
   ✅ 分步骤描述：先创建主界面布局，再设计核心组件，最后添加交互效果
   ```

4. **指定具体的输出格式**

   ```
   ❌ 不指定输出：生成设计
   ✅ 指定输出：生成Figma设计稿、组件库、设计规范文档
   ```

### 问题2：设计不符合无边界原则

**症状**：

- 生成的设计仍然包含传统按钮和菜单
- 界面元素固定，不随用户交互而变化

**解决方案**：

1. **在提示词中明确强调无按钮设计**

   ```
   请明确：不使用传统按钮，通过手势、滑动、拖拽等交互方式触发操作
   ```

2. **提供具体的交互方式**

   ```
   请使用以下交互方式：
   - 手势：滑动删除、拖拽排序、捏合缩放
   - 语音：语音添加、语音删除、语音标记
   - 眼动：注视选择、眼动导航
   ```

3. **参考无边界设计原则**

   ```
   请遵循以下设计原则：
   - 无边界核心：核心功能不依赖固定布局
   - 无按钮设计：不使用传统按钮
   - 一切依托交互：用户行为驱动界面变化
   - 沉浸式体验：用户专注于任务本身
   ```

4. **使用阶段提示词逐步优化设计**

   ```
   先使用阶段1提示词生成概念设计
   再使用阶段2提示词优化界面布局
   最后使用阶段3提示词完善交互设计
   ```

### 问题3：交互效果不流畅

**症状**：

- 生成的交互效果卡顿或不流畅
- 动画效果不自然

**解决方案**：

1. **检查动画实现**

   ```
   请确保动画使用CSS transform和opacity，避免重排重绘
   ```

2. **优化DOM操作**

   ```
   请优化DOM操作，减少重排重绘
   使用虚拟滚动优化长列表性能
   ```

3. **测试性能**

   ```
   请测试性能，使用性能监控工具
   优化动画参数（时长、缓动函数、延迟）
   ```

4. **提供性能优化建议**

   ```
   请提供性能优化方案：
   - 渲染性能：优化DOM操作和重排重绘
   - 动画性能：使用CSS transform和opacity
   - 资源加载：优化图片、字体、脚本等资源的加载
   ```
